Hinweise zur Nutzung

Die API stellt derzeit folgende Funktionen bereit:

  • /getStations: Liefert alle Stationen des Accounts als JSON-String zurück.
  • /getDevices: Liefert alle Geräte des Accounts als JSON-String zurück.
  • /getHouses: Liefert alle Hausobjekte des Accounts als JSON-String zurück.
  • /getMode: Liefert den Status aller Stationen als JSON-String zurück.
  • /getMode/STATION_SERIAL: Liefert den Status der Station mit der Seriennummer STATION_SERIAL als JSON-String zurück.
  • /setMode/MODE: Ändert den Status aller Stationen auf MODE. MODE kann einer der folgenden Werte sein. Die Antwort ist ein JSON-String.
    • away: abwesend
    • custom1: Benutzerdefiniert 1
    • custom2: Benutzerdefiniert 2
    • custom3: Benutzerdefiniert 3
    • disarmed: deaktiviert
    • geo: Geofencing
    • home: zu Hause
    • off: ausgeschaltet
    • schedule: Zeitplan
    • privacyOn: Indoorcam ausschalten
    • privacyOff: Indoorcam einschalten
  • /setMode/STATION_SERIAL/MODE: Ändert den Status der Station mit der Seriennummer STATION_SERIAL auf MODE. MODE kann einer der oben genannten Werte sein. Die Antwort ist ein JSON-String.
  • /getLibrary: Liefert den Link zum letzten Standbild sowie den dazugehörigen Zeitpunkt als JSON-String zurück.
  • /getDeviceImage/DEVICE_SERIAL: Gibt das letzte Bild des Gerätes mit der Seriennummer DEVICE_SERIAL als Bild zurück.

Bitte beachten Sie, dass einige Modi nur mit bestimmten Gerätetypen genutzt werden könnnen. Sollten Sie einen vom Gerät nicht unterstützen Modus setzten wollen, erhalten Sie eine entsprechende Meldung.

Über Skripte können Sie die API abfragen. Dazu haben Sie zwei Möglichkeiten:

  • Möchten Sie die Antwort nicht auswerten, können Sie folgenden Code verwenden, um die Anfrage im Hintergrund (d.h. nicht-blockierend) auszuführen:
    system.Exec("curl --max-time 20 'http://127.0.0.1:52789/setMode/away' &");
  • Möchten Sie die Antwort auswerten, können Sie folgenden Code verwenden:
    string res;
    string err;
    system.Exec("curl --max-time 20 'http://127.0.0.1:52789/setMode/away'", &res, &err);
    Die JSON-Antwort finden Sie im String res.

Bei beiden Varianten werden die entsprechenden Systemvariablen automatisch gesetzt, wenn in den Einstellungen die Option "Systemvariablen bei API Aktionen automatisch aktualisieren" aktiviert wurde.

Verwenden Sie anstelle von 127.0.0.1 die IP-Adresse Ihrer CCU, wenn Sie von Geräten in Ihrem Netzwerk (beispielsweise mit einem Browser) auf die API zugreifen möchten.

Die Angabe max-time 20 bedeutet, dass die Ausführung nach 20 Sekunden abgebrochen wird. Die API-Funktion /setMode kann je Station eine maximale Laufzeit von ca. 10 Sekunden haben. Aus diesem Grund muss der Wert für max-time bei der API-Funktion /setMode entsprechend der Anzahl der Stationen angepasst werden.

Die Zeitstempel der letzten Ereignisse werden nur in folgenden Fällen gesetzt und aktualisiert:

  • Zeitpunkt der Änderung des Modus bei Stationen: die Einstellung "Modus automatisch durch von der Station gesendete Ereignisse aktualisieren" muss aktiviert sein
  • Zeitpunkt des letzten Videoevents bei Kameras: die Einstellung "Pushbenachrichtigungen aktivieren" muss aktiviert sein

Ist die entsprechende Einstellung nicht aktiviert, wird bei Stationen "letzter Statuswechsel: unbekannt" und bei Kameras "letzte Aufnahme nicht verfügbar" angezeigt.
